##lazy关键词
在Swift中，lazy关键字用于延迟初始化一个属性直到它第一次被调用或访问。这意味着，只有在实际需要使用这个属性的时候，它才会被创建和初始化。这种方式通常用于初始化成本较高的对象，或者在对象的构造过程中依赖于某些条件或其他只有运行时才能确定的数据。
使用lazy可以让程序的启动更快，因为它避免了在应用启动时不必要的初始化工作。此外，它还能确保属性只在真正需要时才被创建，从而节省资源。

闭包和函数

定义

  • 函数：函数是具有名称的代码块，可以执行特定的任务。函数可以接收参数、执行操作，并返回结果。函数通常定义在全局作用域或类/结构体的上下文中。
  • 闭包：闭包是可以捕获和存储其所在上下文中的任何常量和变量引用的匿名函数。闭包通常以匿名的形式书写，它可以捕获其定义环境中的值。

捕获上下文

  • 函数：普通函数不能捕获或存储它们所在作用域之外的变量。
  • 闭包：闭包能够捕获周围作用域中的常量和变量，并持有这些值，即使作用域本身已经不存在了，闭包依然可以访问和修改这些值。

书写形式

  • 函数：通常定义得较为明确，具有固定的名称和结构（例如 func doSomething() {}）。
  • 闭包：书写更为灵活，通常用于短小的代码片段，可以直接内嵌于函数调用参数中或者用于变量赋值。闭包的语法结构通常更为紧凑（例如 { (parameters) -> return type in statements }）。

使用场景

  • 函数：更适合执行复杂或多步骤的操作，它们的重用性和结构性更强。
  • 闭包：非常适合用作完成特定任务的代码块，特别是在需要操作函数类型的值时，如回调函数、函数作为参数等场景。 这种方式的好处是代码的灵活性和封装性。不需要为每种行为定义一个单独的函数，也不需要在一个大的事件处理器中使用条件语句来区分行为。每个按钮的行为都是独立定义的，且可以根据需要随时更改，使得代码更加模块化和易于维护。

逃逸闭包

在 Swift 中，当一个闭包作为参数传递给一个函数，并且这个闭包在函数返回之后才被调用，我们称这个闭包为逃逸闭包。换句话说，逃逸闭包是在函数执行完毕后仍然可以在函数外部被调用的闭包。
逃逸闭包主要用于异步操作，比如网络请求、定时器等。在这些情况下，我们需要在函数执行完毕后继续使用这个闭包，比如在网络请求返回数据后更新 UI，或者在定时器触发后执行一些操作。

Person *p1 = [Person new];
Person *p1 会在栈内存中申请一个空间，在栈内存中声明一个Person类型的指针变量p1。p1是一个指针变量，那么只能存储地址。
[Person new]，真正在内存中创建对象的其实是这句代码
第一是在堆内存中申请1块合适大小的空间；
第二是在这个空间中根据类的模板创建对象，类模板中定义了什么属性，就把这些属性依次的声明在对象之中，对象中还有另外一个属性，叫做isa，它是一个指针，这个指针指向这个对象所属的类在代码段中的地址；
第三是初始化对象的属性。如果属性的类型是基本数据类型，那么赋值为0，如果属性的类型是OC的类指针类型，那么就赋值为nil；
第四是返回对象的地址，将这个地址赋值给p1。
因此对象的实际数据（即 Person 对象本身）是存储在堆上的，而指向这些数据的指针（即 p1）是存储在栈上的。

为什么在 Objective-C 中，所有的对象都用一个指针来追踪

  • 很多时候，我们需要把一个对象交给另一个函数或者方法去修改其中的内容（比如说一个 Person 对象的 age 属性），显然如果我们想让修改方获取到这个对象，我们需要的传递的是地址，而不是复制一份。
  • 另一方面，对象的内存是分配在堆上的，而值类型是分配到栈上的。所以一般对象的生命周期会比普通的值类型要长，而且创建和销毁对象以及内存管理是要消耗性能的，所以通过指针来引用一个对象，比直接复制和创建对象要更有效率、更节省性能。

1.2并发

1.2.1定义与调用

在 Swift 中，一个异步函数可以交出它在某个线程上的运行权，这样另一个异步函数在这个函数被阻塞时就能获得此线程的运行权。但是，Swift并不能确定当异步函数恢复运行时其将在哪条线程上。
对于普通的同步函数或方法来说，它们只能运行到完成闭包、抛出错误或者永远不返回。异步函数或方法也能做到这三件事，但同时也可以在等待其他资源的时候挂起。在异步函数或者方法的函数体中，你可以标记其中的任意位置是可以被挂起的。
async 关键字标记某个函数或者方法是异步的
对于那些既是异步又是 throwing 的函数，需要把 async 写在throws 关键字前边。
await标记此处为悬点，，执行会被挂起直到异步方法返回

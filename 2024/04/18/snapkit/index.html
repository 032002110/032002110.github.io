###1.0
No Xcode project found, please specify one
解决方法 应该在包含 .xcodeproj 文件夹的上一级目录中执行该pod init，而不是在 .xcodeproj 文件夹内部。

###1.1
[!] The target demo_12 is declared multiple times.

// Uncomment the next line to define a global platform for your project
// platform :ios, ‘9.0’

target ‘demo_12’ do
//Comment the next line if you don’t want to use dynamic frameworks
use_frameworks!

// Pods for demo_12

target ‘demo_12Tests’ do
inherit! :search_paths
// Pods for testing
end

target ‘demo_12UITests’ do
// Pods for testing
end

end

source ‘https://github.com/CocoaPods/Specs.git‘

platform :ios, ‘10.0’

use_frameworks!

target ‘demo_12’ do
pod ‘SnapKit’, ‘~> 3.0’
end
修改为
// 指定源
source ‘https://mirrors.tuna.tsinghua.edu.cn/git/CocoaPods/Specs.git‘

// 指定全局平台和版本
platform :ios, ‘12.0’

// 使用动态框架
use_frameworks!

// 主target配置
target ‘demo_31’ do
// 主项目依赖的Pods
pod ‘SnapKit’

// 测试target配置
target ‘demo_31Tests’ do
inherit! :search_paths
// 测试依赖的Pods
end

// UI测试target配置
target ‘demo_31UITests’ do
inherit! :search_paths
// UI测试依赖的Pods
end
end

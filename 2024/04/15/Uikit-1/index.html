UIKit Auto Layout 约束

约束激活

  • 约束必须被 激活 (isActive = true) 才能应用到布局中。

避免冲突和不明确性

  • 冲突: 无法同时满足的约束，应该避免。
  • 不明确性: 没有足够信息确定视图位置和大小的状态，应该避免。

约束优先级

  • UILayoutPriority 范围从 1 (最低) 至 1000 (最高)。

translatesAutoresizingMaskIntoConstraints

  • 默认为 true。
  • 使用 Auto Layout 时设置为 false。

锚点

  • 简化约束创建，例如 view.leadingAnchor.constraint(equalTo: superView.leadingAnchor)。

Visual Format Language (VFL)

  • 文字描述布局，能在一行中描述复杂布局。

Interface Builder (IB)

Interface Builder 是 Xcode 中的一个可视化布局工具，它允许开发者通过拖拽的方式来创建用户界面(UI)的约束。以下是 Interface Builder 的一些主要功能：

Resolve Auto Layout Issues

  • Reset to Suggested Constraints
    • 该功能可以重置选定视图的所有约束，并根据 Xcode 的推荐为该视图添加一组新的约束。它会根据视图的当前大小和位置，以及它与父视图或其他视图的关系，生成一组合理的约束。
  • Add Missing Constraints
    • 此功能可以自动为视图添加所需的约束，以确保在不同屏幕尺寸上的一致性。它会根据视图的当前布局推断出缺失的约束，但生成的约束可能不是最优的，因此需要检查生成的约束并加以调整。

Add New Constraints

  • Add New Constraints
    • 你可以为选定的视图添加新的约束，例如设置顶部、底部、左侧、右侧的边距，以及视图的宽度和高度。这些约束有助于确保视图在不同设备和屏幕方向上保持一致的布局。

Align

  • Align
    • Interface Builder 提供了一系列对齐工具，允许你对齐两个或多个视图。例如，你可以使它们的顶部、底部、中心线或基线对齐，以创建一个整洁且一致的用户界面。

安全区 (Safe Area)

  • 考虑了诸如导航栏、标签栏等 UI 元素的边界区域。

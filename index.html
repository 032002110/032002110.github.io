对象的使用

someObject将会收到 doSomething 这条消息。

Person *p1 = [Person new];
Person *p1 会在栈内存中申请一个空间，在栈内存中声明一个Person类型的指针变量p1。p1是一个指针变量，那么只能存储地址。
[Person new]，真正在内存中创建对象的其实是这句代码
第一是在堆内存中申请1块合适大小的空间；
第二是在这个空间中根据类的模板创建对象，类模板中定义了什么属性，就把这些属性依次的声明在对象之中，对象中还有另外一个属性，叫做isa，它是一个指针，这个指针指向这个对象所属的类在代码段中的地址；
第三是初始化对象的属性。如果属性的类型是基本数据类型，那么赋值为0，如果属性的类型是OC的类指针类型，那么就赋值为nil；
第四是返回对象的地址，将这个地址赋值给p1。
因此对象的实际数据（即 Person 对象本身）是存储在堆上的，而指向这些数据的指针（即 p1）是存储在栈上的。

为什么在 Objective-C 中，所有的对象都用一个指针来追踪

  • 很多时候，我们需要把一个对象交给另一个函数或者方法去修改其中的内容（比如说一个 Person 对象的 age 属性），显然如果我们想让修改方获取到这个对象，我们需要的传递的是地址，而不是复制一份。
  • 另一方面，对象的内存是分配在堆上的，而值类型是分配到栈上的。所以一般对象的生命周期会比普通的值类型要长，而且创建和销毁对象以及内存管理是要消耗性能的，所以通过指针来引用一个对象，比直接复制和创建对象要更有效率、更节省性能。

2全局断点


1Swift Error Breakpoint ：swift项目中使用的全局断点，当添加此断点后，程序会暂停在使用throw抛出异常的代码处。
2Exception Breakpoint：用来捕获程序中的异常，当程序发生数组越界，设置了非空参数为nil等问题会停留在异常处。
3OpenGL ES Error Breakpoint ：当OpenGL ES发生错误时，程序会停留在错误处。

5Runtime Issue Breakpoint:运行时问题断点：当运用运行时方法出现错误时、程序会暂停在出错处。
6constraint Error Breakpoint是约束错误断点，自动布局控件出现约束错误时，添加这个断点后，程序会暂停在出错处，打印错误信息。
7Test Failure Breakpoint 是测试错误断点，当出现测试没有通过情况时，添加此断点会，停在失败的测试处。

  • 适用场景
    – 与多个不同系列相关产品交互
    – 有一个基于一组抽象方法的类
    例子
    定义了一个区分用户要登入还是注册的枚举类
    定义了authview工厂协议，两个静态方法要求 1️⃣根据是登入还是注册返回对应authview
    2️⃣根据登入还是注册返回auth控制器
    定义了两个类学生authview工厂和教师authview工厂都实现了authview工厂协议
    学生工厂第一个静态方法返回学生登入view或学生注册view
    教师authview工厂返回教师登入view或学生注册view
    定义了authview协议，该协议定义了视图接口用于处理授权操作，要求实现视图内容
    可选的authaction闭包用于处理授权操作，描述视图
    教师/学生 登录/注册视图这四个类实现了authview协议，这四个类都有authHander属性，这是一个可选闭包
    用于处理视图中的按钮动作或其他按钮，可以用闭包执行相应操作
    定义学生/教师认证视图控制器继承认证控制器
    定义一个私有类负责管理认证控制器的展示

分析一下：这个例子中AuthViewFactory充当了抽象工厂的角色，负责AuthView和AuthVIewController实例生产
学生和教师各自登入和注册视图构成不同产品族，学生认证工厂和教师认证工厂就是具体的工厂。替补方法AuthView和
AuthViewController类确保了学生和教师认证视图和控制器遵循相同规范。私有类作为客户端不直接依赖于具体产品类

交叉熵损失（Cross-Entropy Loss）：
交叉熵损失是分类任务中最常用的损失函数之一。
它通过衡量预测概率分布与真实概率分布之间的差异来优化模型。
在自然图像分类或识别任务中，交叉熵损失通常能够取得良好的效果。
Focal Loss：
Focal Loss 是一种设计来解决类别不平衡问题的损失函数，特别是在目标检测领域如RetinaNet中使用。
如果自然图像数据集中存在类别不平衡问题（例如，背景类别远多于目标类别），Focal Loss 可以帮助模型更关注于少数类别，从而提高识别性能。

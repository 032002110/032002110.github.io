Moya

Moya是对Alamofire的封装，Moya 将网络请求封装为 TargetType 协议的实现，每个 endpoint 对应一个 enum 的 case

##先定义TargetType协议的枚举来配置网络请求

  • TargetType 是 Moya 的核心，每个网络请求都基于它来配置。它要求定义几个基本但关键的属性：
  • baseURL （基础URL）：所有请求的基本URL。
  • path （路径）：与 baseURL 结合，形成完整的请求URL。
  • method （方法）：HTTP请求的方法，如 GET、POST 等。
  • task （任务）：定义请求的具体内容，如是否有参数，参数是怎样编码的，是否上传文件等。
  • task 是 TargetType 中最复杂的部分，它定义了发送请求时的具体行为和数据。task 有几种类型，每种类型适用于不同的网络请求场景：
    – requestPlain：发送一个基本的请求，不带任何参数。
    – requestData(Data)：允许你发送一些二进制数据作为请求体。
    – requestParameters(parameters: [String: Any], encoding: ParameterEncoding)：使用这种类型可以发送带参数的请求。参数可以是 .url、.json、.formData 等格式编码。
    – requestCompositeData(bodyData: Data, urlParameters: [String: Any])：允许在同一个请求中同时发送 URL 参数和请求体数据。
    – uploadMultipart([MultipartFormData])：用于上传文件或数据时，可以构建复杂的多部分请求体。
  • headers （头部）：为请求指定 HTTP 头部。 Moya 返回的 Response 对象包含了完整的 HTTP 响应数据。你可以使用 Moya 的扩展方法来简化数据处理，如：
  • response.mapJSON(): 将响应数据映射为 JSON 对象。
  • response.map(): 将响应数据映射为指定的数据模型，通常需要遵循 Decodable 协议
    使用Moya的provider来发送请求： 使用插件

##lazy关键词
在Swift中，lazy关键字用于延迟初始化一个属性直到它第一次被调用或访问。这意味着，只有在实际需要使用这个属性的时候，它才会被创建和初始化。这种方式通常用于初始化成本较高的对象，或者在对象的构造过程中依赖于某些条件或其他只有运行时才能确定的数据。
使用lazy可以让程序的启动更快，因为它避免了在应用启动时不必要的初始化工作。此外，它还能确保属性只在真正需要时才被创建，从而节省资源。

对象的使用

someObject将会收到 doSomething 这条消息。

Person *p1 = [Person new];
Person *p1 会在栈内存中申请一个空间，在栈内存中声明一个Person类型的指针变量p1。p1是一个指针变量，那么只能存储地址。
[Person new]，真正在内存中创建对象的其实是这句代码
第一是在堆内存中申请1块合适大小的空间；
第二是在这个空间中根据类的模板创建对象，类模板中定义了什么属性，就把这些属性依次的声明在对象之中，对象中还有另外一个属性，叫做isa，它是一个指针，这个指针指向这个对象所属的类在代码段中的地址；
第三是初始化对象的属性。如果属性的类型是基本数据类型，那么赋值为0，如果属性的类型是OC的类指针类型，那么就赋值为nil；
第四是返回对象的地址，将这个地址赋值给p1。
因此对象的实际数据（即 Person 对象本身）是存储在堆上的，而指向这些数据的指针（即 p1）是存储在栈上的。

##自动引用计数
概念：在 LLVM 编译器中设置 ARC(Automaitc Reference Counting) 为有效状态，就无需再次键入 retain 或 release 代码。

操作 oc方法
生成并持有对象 alloc/new/copy/mutableCopy
持有对象 retain
释放对象 release
废弃对象 dealloc
**对象创建一般需要调用alloc和init两个方法。alloc为对象分配内存，然后此时对象并没有初始化，所以不能正常工作。init方法调用后，对象被初始化，此后才可以正常工作。
MyClass myObject = [[MyClass alloc] init]; 最常见实例化过程*

为什么在 Objective-C 中，所有的对象都用一个指针来追踪

  • 很多时候，我们需要把一个对象交给另一个函数或者方法去修改其中的内容（比如说一个 Person 对象的 age 属性），显然如果我们想让修改方获取到这个对象，我们需要的传递的是地址，而不是复制一份。
  • 另一方面，对象的内存是分配在堆上的，而值类型是分配到栈上的。所以一般对象的生命周期会比普通的值类型要长，而且创建和销毁对象以及内存管理是要消耗性能的，所以通过指针来引用一个对象，比直接复制和创建对象要更有效率、更节省性能。

2全局断点


1Swift Error Breakpoint ：swift项目中使用的全局断点，当添加此断点后，程序会暂停在使用throw抛出异常的代码处。
2Exception Breakpoint：用来捕获程序中的异常，当程序发生数组越界，设置了非空参数为nil等问题会停留在异常处。
3OpenGL ES Error Breakpoint ：当OpenGL ES发生错误时，程序会停留在错误处。
4Symbolic Breakpoint ：符号断点，这是最强大的一种全局断点，系统的方法也可以断住.
5Runtime Issue Breakpoint:运行时问题断点：当运用运行时方法出现错误时、程序会暂停在出错处。
6constraint Error Breakpoint是约束错误断点，自动布局控件出现约束错误时，添加这个断点后，程序会暂停在出错处，打印错误信息。
7Test Failure Breakpoint 是测试错误断点，当出现测试没有通过情况时，添加此断点会，停在失败的测试处。

1.1 抽象工厂模式

声明该系列中所有产品构造方法的接口，基于抽象工厂接口创建不同的工厂类

  • 适用场景
    – 与多个不同系列相关产品交互
    – 有一个基于一组抽象方法的类
    例子
    定义了一个区分用户要登入还是注册的枚举类
    定义了authview工厂协议，两个静态方法要求 1️⃣根据是登入还是注册返回对应authview
    2️⃣根据登入还是注册返回auth控制器
    定义了两个类学生authview工厂和教师authview工厂都实现了authview工厂协议
    学生工厂第一个静态方法返回学生登入view或学生注册view
    教师authview工厂返回教师登入view或学生注册view
    定义了authview协议，该协议定义了视图接口用于处理授权操作，要求实现视图内容
    可选的authaction闭包用于处理授权操作，描述视图
    教师/学生 登录/注册视图这四个类实现了authview协议，这四个类都有authHander属性，这是一个可选闭包
    用于处理视图中的按钮动作或其他按钮，可以用闭包执行相应操作
    定义学生/教师认证视图控制器继承认证控制器
    定义一个私有类负责管理认证控制器的展示

分析一下：这个例子中AuthViewFactory充当了抽象工厂的角色，负责AuthView和AuthVIewController实例生产
学生和教师各自登入和注册视图构成不同产品族，学生认证工厂和教师认证工厂就是具体的工厂。替补方法AuthView和
AuthViewController类确保了学生和教师认证视图和控制器遵循相同规范。私有类作为客户端不直接依赖于具体产品类
降低系统耦合度。当需要增加新的用户类型，可以只增加新的具体工厂和产品类，保证了扩展性。
